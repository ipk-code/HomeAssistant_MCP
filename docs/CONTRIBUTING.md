# Contributing

## Workflow

1. Create or update tests before changing behavior.
2. Keep changes minimal and domain-focused.
3. Run `python3 -m unittest discover -s tests -p "test_*.py"` before every commit.
4. Review code for security issues before every commit.
5. Update repository documentation when behavior, APIs, or setup changes.

## Development Rules

- Do not add raw Lovelace card dictionaries to the public tool API.
- Do not accept direct file paths from MCP clients.
- Keep dashboard mutation paths constrained to validated metadata, views, and cards.
- Preserve optimistic concurrency checks for state-changing operations.

## Commit Expectations

- Use concise commit messages that describe the purpose of the change.
- Include tests and documentation in the same commit when they are part of the feature.
- Do not commit generated bytecode, secrets, or local environment files.

## Review Checklist

- Inputs are validated and unknown fields are rejected.
- Errors do not expose internal stack traces to MCP clients.
- File writes are atomic and leave no partial state on failure.
- Test coverage includes success paths and failure paths.
- Documentation matches the implemented behavior.
